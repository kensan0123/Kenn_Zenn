## 導入
Zenn CLIは、Zenn.dev への記事・本の作成・公開をローカル環境から効率的に行うための公式ツールです。本稿では、Zenn CLIの基本的な使い方と実践的なワークフローを詳しく解説します。

## 背景
Zenn では、記事はMarkdown形式で作成され、公開のためのワークフローを整備することが望まれます。CLIを用いると、テンプレートの生成・ローカルプレビュー・公開準備を一元化できます。特に以下の点が利点です。
- テンプレートを素早く生成
- ローカルプレビューで仕上がりを確認
- 複数人での編集時のファイル配置が統一される

Zenn CLIの導入により、エディタの環境に依存せず、統一されたファイル構成とビルド結果を手元で検証できるため、品質向上とリリースサイクルの短縮が期待できます。

## 手順
### 1) インストール
```bash
npm i -g @zenn/cli
```

### 2) 初期化とテンプレート作成
```bash
zenn new:article
```
このコマンドを実行すると、作成先ディレクトリに記事の雛形が生成されます。生成物は通常、Markdown形式のファイルと必要なメタ情報を含む front matter ヘッダです。雛形には、タイトル・型（Tech/Idea）・カテゴリ（topics）などのメタ情報を設定済みの状態で配置されます。

### 3) 記事の編集
生成された markdown ファイルを開いて、タイトル、本文、カテゴリ（topics など）を編集します。例:

```yaml
---
title: Zenn CLIの使い方
type: Tech
topics:
  - zenn
  - cli
---

# Zenn CLIの使い方

この記事では、基本的な使い方と実践的なノウハウを解説します。
```

### 4) ローカルプレビュー
```bash
zenn preview
```
このコマンドでローカルのプレビューサーバが起動します。ブラウザでローカルのプレビュー URL にアクセスして、記事の表示を確認します。

### 5) 追加の機能と運用
- 記事の一括作成: 複数の記事をテンプレートから作成する場合、コマンドラインオプションやスクリプトを活用する方法もあります
- 自動化: CI/CD と組み合わせて、リポジトリの push をトリガーに自動ビルド・プレビューを実現する

### 6) 公開とデプロイ
Zenn への公開は、通常はリポジトリの公開手順と連携します。Zenn公式サイトの連携手法を参照してください。CLI だけで完結する公開フローは環境依存です。実運用では、プッシュ後のマージ／タグ付けなどの運用ルールを決めておくと安定します。

### 実用のコツ
- テンプレートの命名規約を統一する
- front matter に必要なメタ情報を揃える
- 記事内のコードブロックには適切な言語名を付ける
- 複数人で執筆する場合はブランチ運用と見出しの命名規約を統一する

## まとめ
Zenn CLIを活用すると、記事作成・プレビュー・公開の一連の作業をローカル環境で完結できます。本稿の手順をベースに、チームのワークフローに合わせてカスタマイズしてください。
